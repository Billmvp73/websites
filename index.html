Scene Caching for Video Capture Data Reduction



Key words: Machine Vision, Data Reduction, Intelligent Sampling, Motion Compensation, Motion Segmentation, Background Modeling, Encoder-Decoder

  • Developed an intelligent Scene Cache that uses heterogeneous spatial capture, temporal buffering and multiround capture to reduce the amount of data captured and transferred without accuracy reduction.

    • Multiscale Deep Attention-Sampling Model



    Key words: Multiple Instance Learning, Attention Sampling, Multiresolution, Object Detection

    Existing deep architectures cannot operate on very large signals such as megapixel images due to computational and memory constraints. To tackle this limitation, Angelos Katharopoulos and Franc ̧ois Fleuret propose a fully differentiable end-to-end trainable model that samples and pro- cesses only a fraction of the full resolution input image. They refer to their method as attention sampling and it can process images of several megapixels with a standard single GPU setup. However, we found their method was only able to learn the distribution of simple, small objects because their architecture is trained with a small, fixed-size patch configuration. Our work introduces a multiresolution architecuture, combine it with attention sampling methods and change the way to sample patches from the multiscale sample space so that our network is able to learn the attention distrbution of complicated, size-variant objects like vehicle without large inference overhead.

    Faculty Mentors: Robert Dick

    Web Automation based on Programming-by-Demonstration



    Key words: Programming-by-Demonstration, Web Automation, Recorder, Domain Specific Language

    Problem Description: Web automation is the concept of letting software robots perform actions, tasks, and processes on a web browser or web application. In daily life, when people interact with web pages, including filling out forms or fields, scraping data from web pages, clicking buttons and elements or shifting among pages, they often have to do repetitive operations. Our goal is to automate those repetitive tasks. The idea is, when a user is manually performing the task, our tool is able to record and analyze these actions and suggest next action that the user is likelyto perform. The user can simply approve our suggestion or manually correct the action if our suggestion is wrong.


    Contribution:

  • Designed the specific context grammer for this project and a corresponding domain specific language including features like string parsing, arithmetic operation, conditional and loop logics
  • Used Selenium IDE to generate ground truth programs and implemented a recorder to record real-time operations and a user interface to allow users to correct our synthesized program using Javascript
  • Designed the high-level algorithm to raise the level of abstraction at which users' actions are specific and synthesize programs.
  • Implemented the algorithm for loop-structed program synthesis
